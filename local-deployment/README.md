# üìÅ Local Deployment

This directory contains Docker Compose files for local development of Server Info Watchdog.

## üìã Compose Files

| File | Purpose |
|------|---------|
| `docker-compose.yml` | Run the watchdog service locally |

## üöÄ Usage

### Start Watchdog

```bash
# From project root
docker compose --env-file .env -f local-deployment/docker-compose.yml up --build
```

### Stop Services

```bash
docker compose --env-file .env -f local-deployment/docker-compose.yml down
```

### View Logs

```bash
docker compose --env-file .env -f local-deployment/docker-compose.yml logs -f
```

## üìù Configuration

Make sure you have a `.env` file in the project root with:
- `TELEGRAM_BOT_TOKEN` - Telegram bot token from BotFather
- `SERVER_NAME` - Name to identify this server in notifications
- `ERROR_CHAT_IDS`, `WARNING_CHAT_IDS`, `INFO_CHAT_IDS` - Telegram chat IDs
- Threshold settings for CPU, memory, disk, etc.

See `setup/.env.template` for all options.

## üìÇ Server Info JSON

The watchdog reads JSON files from the `serverInfo/` directory. These should be generated by the `server-info` CLI tool running on each server you want to monitor.

For local testing, you can create a sample `serverInfo/system_info.json` file.
